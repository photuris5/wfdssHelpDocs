WFDSS v1.0.5 Release Notes (08/14/2025)

Fixes

Analysis General

  • Fixed a bug causing the ERC Classes and Stream data to not be retained after an FSPro status is changed from Review.

  • Fixed a bug reading the Slope Class for RAWS stations properly.

  • Fixed a bug preventing users from marking the Analysis Summary “Ready for Publication” when there were expired analyses that were not included in the decision.

  • Fixed an issue in which an FSPro analysis showed as "Failed" if there was a network error while it was processing; the system will now retry to connect to the Fire Modeling Service Framework.

FSPro

  • The default wind dates in FSPro were adjusted to 2 weeks prior to the start date and 4 weeks after the start date. Analysts can set the date range for any length of days.

IRWIN Writing

  • Updated the IRWIN Observer Links in the Support Dashboard to the correct IRWIN environments.

Map Layers

  • Ensured all the Snag Hazard data categories are current and reflected in the Firefighter Safety Chart.

Strategic Course of Action

  • Added Suppression Strategy to the CSV Download on the Strategic Course of Action tab.

WFDSS v1.0.4 Release Notes (08/01/2025)

Features/Improvements

Analysis General

  • Updated the application to connect to the Fire Modeling Service Framework (FMSF) Version 2.0.

FSPro

  • Ensures all the FSPro outputs (burn probability, average flame length, average arrival time) properly display on the map with FMFS V2.

WFDSS v1.0.3 Release Notes (07/28/2025)

Fixes

Analysis Actions

  • Ensured that Analysis Notes are saved and viewable in Analysis Requests.

  • Fixed an error causing analysis failures when a user deleted a cell value in the Winds Matrix of an FSPro.

Analysis General

  • Ensured the ERC Class data remains visible on the Weather Tab after an FSPro run is submitted.

  • Fixed an issue in which fuel moisture values in the FSPro ERC Classes were not consistent; initially using 1300 observation values and subsequently displaying all hours observation values.

  • Fixed a bug that labeled landscapes with having "No Rules" even when the landscapes did have edit rules applied on the Run/Results subtab of an Analysis.

  • Fixed a bug that gave an Incident Editor a Red Error message when requesting an FSPro run.

  • Fixed an issue that caused user edited values for ERCs and Winds in FSPro to sometimes default to the original values.

Decision General

  • Fixed an issue that allowed for editing of the Organization Assessment while the Relative Risk section was not marked Ready for Publication.

FSPro

  • Fixed an issue causing the RAWS stations selected in one incident to pre-populate in a subsequent incident.

  • Fixed an issue to ensure ERC Class values edited in the user interface are used in the modeling service.

  • Fixed an issue in which an additional day was added to the start date of an FSPro analysis if a forecast was not used.

  • Fixed an issue that prevented the RAWS station list for FSPro modeling from updating to the incident's location if an analyst navigated between incidents in one session.

  • Fixed a bug that sometimes prevented the Wind data in an FSPro run from being retained.

  • Fixed an issue in which the dates selected for FSPro Winds generation would default to the whole year after the user generated the Wind Matrix.

Landscape Rules

  • Fixed a bug that caused LCPs to fail if there were many landscape edits.

Map Layers

  • Fixed a bug preventing the downloading of shapes that had a colon in the shape name.

Map Performance

  • Fixed a bug causing the Print Map feature to not function if the map has more than one FSPro Flame Length displayed.

PDF Generator

  • Fixed an issue in which the Recommended Organization in the PDF decision did not always match the User Interface in the application.

  • Fixed a bug that copied the verbiage and text entered into the Fuel Conditions text box of Relative Risk to the Infrastructure/Natural/Cultural Concerns text box in the PDF of the decision.

  • Fixed an issue in which an unpaired quote in the language associated with Spatial Fire Planning Shapes caused corrupted decisions.

UI Basics

  • Subtabs on the Support Tab used by Support Role now identify which tab the user is on.

Features/Improvements

508 Compliance & UI Standards

  • Tested 508 compliance in the following areas of the application: Organization Assessment and Complexity, Decision Approval & Assessment.

  • Assessed the following areas for 508 compliance: Incident Info, Analysis-New Analysis, Existing Analyses, and Strategy.

Analysis Actions

  • Ensured Analysis Notes remain with Analysis Requests.

Analysis General

  • Fixed an issue causing a note associated with an analysis request to not save consistently.

  • Added a link to the LANDFIRE website in the Landscape tab for analysis to facilitate verification of availability of ModFis fuel model data.

  • Ensure the Current Year ERC values after the Start Date of an FSPro are not included in the Input File.

Chart and Image Integration

  • Ensures the Windrose Chart in FSPro is properly updated when users change weather parameters.

Cleanup

  • Corrected the "Mimic Role" function that Support Role uses to ensure proper functionality.

Decision Approval Assessment

  • Made the Incident Info Tab read only when an Incident is declared out.

FSPro

  • The Weather Tab of the FSPro model has been refactored to improve user experience and performance.

  • Updated the FSPro Input file to be compliant with FMSF 2.0.

  • Adjusted the User Interface of FSPro ERC Classes to meet the requirements of FMFS 2.0.

  • Updated the Analysis Results tab to list the FMFS ID number under the status accordion.

  • Updated the FSPro minimum ERC Stream to allow a zero value.

  • Adjusted the input file for FSPro to ensure only integers are used for ERC to support changes in the Fire Modeling Service Framework.

Map Layers

  • Updated the Snag Hazard to include all categories and ensured it populates correctly in the Spatial Inventory.

  • Updated the URL for Infrastructure - Utilities and Transportation.

  • Updated the service URL for Incident Strategic Alignment Process (ISAP).

Technical Debt Cleanup

  • Made adjustments to improve diagnosing errors and other failures by capturing needed details in the logging.

UI Basics

  • The column header for Recommended Organization was updated to Unit Recommended Organization on the My Home tab.

  • Users receive a warning message if they try to navigate away from the User Profile tabs without saving their work.

  • Users receive a warning message if they try to navigate away from the Strategy tabs without saving their work.

  • Updated the warning message users receive if they navigate off a page with unsaved changes so that is it more clear what action to take in order to save edits vs. abandon changes.

  • Bulleted Lists have been re-enabled; PDFs of the Decision content will now function with bullets.

  • Users receive a warning message if they try to navigate away from the Decision Approval/Assessment tabs without saving their work.

  • Users receive a warning message if they try to navigate away from the Organization Assessment tabs without saving their work.

  • Users receive a warning message if they try to navigate away from the Relative Risk tab without saving their work.

  • Users receive a warning message if they try to navigate away from the Analysis tabs without saving their work.

  • The button used to create and download shapes has renamed to “Download” to make it more clear for users.

User Role Access

  • Fixed a bug allowing Editors who are Non-Collaborators to switch On/Off the Ready for Publication toggles.

  • Ensured Editor, non-Collaborator users are not able to create 'Analysis' or 'Request for Analysis'.

WFDSS v1.0 Release Notes (03/26/2025)

The WFDSS NextGen version 1.0 application has been released and includes minimum viable product for searching for an incident, starting a decision, creating an FSPro Analysis, entering decision information and creating a decision document.  The major feature areas included in this release are as follows:

  • FamAuth roles and Authentication login integration
  • My Home
  • Support Dashboard
  • User Support
  • User Profile
  • User Search
  • Incident Browsing
  • Incident Info
  • Incident Decision
  • Interactive map with outside map layers
  • WFDSS Initiated Incident Points
  • Map Tools
  • Map Layer List
  • Print Map
  • Planning Area
  • Ignition Shapes
  • Mask Shapes
  • Shape Downloads
  • Shapefile Upload Tool
  • Spatial Inventory
  • Landscape Extent
  • Fire Perimeter
  • Barrier Shapes
  • Incident and Analysis shapes saved to NIFC
  • Ck 5 Rich Text Editor
  • Analysis Summary
  • Create/Request FSPRO Analysis
  • LANDFIRE landscape submission
  • Landscape rules
  • FSPRO Results downloads
  • Analysis Notes
  • Strategic Course of Action
  • Strategic Assessment
  • Cost
  • Relative Risk
  • Organizational Assessment
  • Decision Rationale
  • Publishing Decision
