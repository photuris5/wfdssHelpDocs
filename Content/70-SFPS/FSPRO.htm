FSPRO

FSPro Overview

About FSPro

FSPro is a geospatial probabilistic model that predicts fire growth; and is designed to support long-term decision-making (more than 5 days).

FSPro calculates two-dimensional fire growth and maps the probability that fire will visit each point (cell) on the landscape of interest under the following conditions:

  • during the specified time

  • in the absence of suppression

  • based on the current fire perimeter or ignition point

    FSPro has limited ability to model suppression effects, though users can limit surface fire spread with the use of barriers and masks.

    FSPro is valuable to fire managers because it can help address uncertainty in long-term fire behavior analysis where weather forecasts are unreliable. It can help support risk-based decision making when the results are combined with information about resources and assets (values to be protected).  FSPro can provide insight for strategic decision-making by helping answering questions such as the following:

  • What is the probability a fire will reach a point of concern on the landscape during a specific period of time (such as within 7, 10, 14 or more days)?

  • What is the probability that a fire will reach an administrative boundary during a specific period of time?

  • Which fires or segments of fires will likely need suppression action?

  • How large might the fire get?

  • What is the expected fire size distribution?

  • How soon might the fire reach critical points of concern? (beta version [in testing])

FSPro Considerations

Like all model systems, FSPro has numerous considerations, assumptions and limitations specific to each model it uses.  It is important to be familiar with these when viewing model results.  FSPro uses the same underlying fire models as BehavePlus, FARSITE, and FlamMap. The assumptions and limitations of those models are also inherent in FSPro (e.g., uniform fuels, etc).  Some additional assumptions and limitations of FSPro include the following:

  • FSPro results assume no suppression action (other than the inclusion of barriers to simulate effective fireline construction).

  • Limited fine-scale temporal variability in weather. This means that the weather is constant for the entire day (1 ERC value and related fuel moistures, 1 wind speed and wind direction).

  • The peak burning period is assumed because the ERC, fuel moisture, and wind are obtained at that time.

  • There is no correction of fuel moisture for elevation or aspect (forthcoming).

  • The FSPro model uses 100% for foliar moisture content. This value cannot be edited.

  • Winds and fuel moisture are independent.

  • No climate change prediction is available (assumes historic climate).

  • The extremely rare event may or may not be represented by the simulation.

  • The resulting burn probability maps are easily misinterpreted as a fire progression, such as in FARSITE (FSPro results show probability contours NOT daily progression perimeters!).

  • Model output is contingent on model input and modeler expertise. FSPro can only be as accurate as the data used as inputs to the model. The following two data sources should be critiqued:

    • Landscape: Needs to be up to date (often the landscape will need to be edited to provide realistic modeling results); use of the landscape editor might be needed.

    • RAWS: One or two can be selected and need to be representative of the analysis area for both ERC and wind values.

Why Model Using FSPro instead of FARSITE?

FSPro is valuable to fire managers because it can help address uncertainty in long-term fire behavior analysis where weather forecasts are unreliable. FSPro uses both weather forecast and probabilistic weather inputs to provide a long-term modeling solution. It can help support risk-based decision making when the results are combined with information about resources and assets (values to be protected).  FSPro can provide insight into strategic decision-making.

Fire Area Simulator (FARSITE) (Finney 1998) is a fire simulation model used over a short period of time (1-5 days) where there is good confidence in the predicted weather. If we want to get an idea as to where and when a fire might spread in a few days using forecasted weather, FARSITE is the appropriate application to use.  When a reliable weather forecast is used, FARSITE is useful for answering questions such as the following:

  • What is the predicted fire size and shape over the next 1 to 5 days?

  • What are the likely fire behavior characteristics within the predicted fire perimeter?

  • When will the fire reach specific points of concern?

Tutorial

How to Run FSPro

To run FSPro users with the Fire Behavior Analyst role will select the Analysis tab near the top of the screen (users must first already be viewing an incident AND the incident must already have a decision started). Under the heading New Analysis select FSPro from the Analysis Type and click Create Analysis.

Name the analysis, which will default to the current date, but can be changed to something more descriptive. Ensure the Analysis Start Date is correct, which will default to the current date. Add Analysis Notes if desired. The notes can include information about fire behavior, fuels, weather and burning conditions, the purpose of the modeling, etc. Select Save Analysis.

The following screen provides tabs the analyst uses to set up the model and include: FSPro (general run information), Landscape (for defining and editing the LCP), Weather (for selection and editing of ERC and Wind inputs), Run/Results (has a checklist for all required inputs before submission, download options, status options).

About FSPRO General Information Section

The general information items listed below are required to run an FSPro analysis. Once entered, click Save.

  • Analysis Name – Required user determined to identify analysis run. Use a descriptive name to help find analysis later.

  • Analysis Starte Date - Required user defined start date of the analysis.

  • Number of Fires – Required. The more fires the analyst chooses yields an improved probabilistic result. Less fires are highly recommended for initial runs.

  • Number of Days - Required user defined number of days to run the model. Analysts should consider the number of forecast days, and the number of probabilistic weather days needed for the analysis. The combination of forecast and probabilistic yields the number of required days for the run. Typically, the minimum number of days is 7 since the intent of the model is usually to look longer term than the near-term forecasts provide.

  • Crown Fire Method – Required user can choose from Scott/Reinhardt (2021) or Finney (1998). Scott/Reinhardt tends to yield more robust spotting and crowning. Scott/Reinhardt.

  • Ignition File – Required, considering using the hot spots with potential for growth verses the entire perimeter. Ignition can be quickly drawn on the map with the draw tool. (See How to Create/Draw Ignitions and Barriers)

  • Barrier File – Optional, shapefile intended to be a barrier to fire spread such as a fuel break, road or previously burned/treated area. Barrier shape cannot be a line.

  • Check/Uncheck Fill Barrier – Optional and only applicable to a barrier that is a polygon- fill the barrier to prevent spotting into the interior of the barrier shape. If spotting is desired, do not fill.

Landscape

The Landscape subtab is used to create and edit landscapes. The landscape extent is defined by the analyst who determines the size of the landscape file needed for fire behavior modeling. All fire behavior analysis outputs are limited to the area included in the landscape extent. If you find that the extent is too small, create a larger extent. For FSPro, FlamMap is run for all cells contained by the landscape extent, so it is important to limit it to a reasonable size. The landscape extent needs to be large enough to contain fire behavior outputs, but not so large that it slows processing time.

Analysts must define the landscape extent before a landscape is created, and you should create a new extent if FSPro outputs are larger than the extent used. When creating the landscape extent, consider the following:

  • The larger the extent, the longer the analysis will require for processing.

  • The extent should contain your ignition point or polygon and should encompass projected fire growth for the duration of your fire simulation.

  • Resolution can range from 30 meters to 420 meters with 30meter increments and for FSPro defaults to 90 meters.

  • Resolution can affect the processing time of the analysis. Finer resolution has more pixels and will take longer to process since the model runs for each pixel.

  • 40 fuel models or 13 fuel models can be chosen, the default will be the 40 fuel models.

  • The data Source list of landscape sources will not include all versions of Landfire (LF) products will be available. As newer versions are released, older versions will be removed. The base mapped versions (LF 2001/ 1.0.5 and LF 2016 Remap / LF 2.0.0) will always be available via the LFPS; however, only the three updated versions (e.g., LF 2012 / 1.3.0) will persist in LF Product Service (LFPS).

  • WARNING: The Fire Modeling Service will not accept landscape extents that exceed 2,500,00 cells, so while you can draw and save a very large extent, if the resultant cells based on the resolution exceed the 2,500,000 you will get an error message directing you to decrease the extent or increase the resolution.

Weather

ERC Classes

        

ERC classes, once generated for the selected weather station, define the range of historical weather data that help drive the FSPro model. The ERC classes are comprised of historical daily ERCy values for a date range specified by a Fire Behavior Specialist (FBS).

 

The daily ERC values are grouped into ranges of ERC values called bins. WFDSS assigns each bin an associated percentile value as well as burn period and spotting probability/delay information. Dead fuel moisture values are derived from NFDRS. Live fuel moistures are derived from the Growing Season Index (GSI). The model uses these binned values to generate the time series analysis. A forecasted ERC value OR an auto-correlated ERC value dictates which set of binned values are used to calculate fire spread for any given day.

 

Generally, the default bin values are adequate, but analysts may want to make changes in some situations. If current or predicted daily ERC values are less than the lowest bin displayed (60th percentile is the default lowest bin):

  • No modeled fire growth occurs on those days, or

  • If fire is actively spreading, you may need to add additional rows to the ERC classes table.

If current or predicted daily ERC values are greater than the highest bin displayed (97th percentile bin typically), there is no need to add rows above the highest bin. FSPro does not recognize a 'ceiling' and models fire growth using values in the highest bin for those days.

 

Selecting a Weather Station for ERC Classes (FSPro)

In the FSPro model, fuel moisture is binned into Energy Release Component (ERC) percentile classes based on data from the selected weather station. FSPro uses ERC (fuel model Y) to calculate and bin the live and dead fuel moisture inputs. Analysts need to take the time to select a weather station that will best capture this type of information based on elevation, proximity to the fire location, and the ERC Class outputs based on the selected start year and beginning/end months. It is important to pay attention to elevation and locate a station with similar weather (choose a station within +/- 1000 to 2000' elevation) but it is not necessary to find a station with similar drainage alignment or to try to capture other localized wind influences when selecting a weather station to use for ERC.

 

The populated list of available stations weather stations from the Fire Environment Mapping System (FEMS) dataset.

 

Station density and location will not always be optimal, and in some instances, the closest station may not be the best choice. Evaluate the station information and always select the most representative station for the fire and location.

Climatology Date Filter

The data filter Has 5 required fields:

  • Start year

  • End year

  • Start Month/Day

  • End Month/Day

  • Fuel Model

These dates, selected by a Fire Behavior Specialist, determine the range of days and corresponding data that will be used to generate ERC percentile classes. The default date range is based on the Geographic Area and should be evaluated and adjusted as needed by the FBS. When setting a date filter, you can modify the start year, but the end year is always the current year. When modifications are made to either the Station Information or the Date Filter, always click the Generate ERC Classes button to update the ERC table. The date filter is one of the most important station considerations when selecting a weather station for generating ERC percentile classes because it will determine if the station has suitable weather data for the time-period you are interested in modeling. A minimum of 10 years of data is recommended, but it's important to ensure there are no large gaps in the data, especially if the station has a very long record.

 

The actual weather data available for an individual station is unique to each station. The number of years, months or days within a specific year that actual weather data exists cannot be determined in WFDSS and it may be more appropriate to determine date filter values outside the WFDSS system. You can view station data and graphs in FEMS. You can also download a .FWX file for the selected weather station from FEMS for closer examination in Fire Family Plus to help you determine if sufficient data exists for a specific time-period.

ERC Stream

The ERC Stream Menu Option provides opportunities for Fire Behavior Specialists to manage the weather stream and forecast data for an analysis. Here analysts can complete the following tasks:

  • View and/or modify the ERC Stream for the weather station selected and time-period specified on the ERC Classes Menu Option.

  • Get National Weather Service (NWS) forecast data for inclusion in an FSPro simulation.

  • Add rows to or delete rows from the weather forecast.

  • Manually add and/or modify forecast data.

The ERC Stream populates and is saved automatically when an analyst generates and then saves ERC Classes on the ERC Classes page and is for the time-period specified in the ERC classes' date filter.

NOTE: Currently if there are gaps in data, the end user will need to manually fill in the gaps. Users can view the ERC data being retrieved from FEMS.

NOTE: Keep in mind that the ERC’s and wind information entered in the forecast section will be used for each of the designated days for every fire simulated in the FSPro analysis. For instance, if you used forecast data for July 27th and 28th for a 10-day run and you are simulating 1000 fires, each of those fires will use the forecast data for July 27th and July 28th and then historic weather will be used to generate artificial seasons for the remaining eight days. If the forecasted ERC is in the 90th percentile bin and you have given that bin a 5 hour burn period, it will use the information from that bin (fuel moistures, burn period, spot probability and delay) as well as the forecasted wind speed and wind direction to simulate fire spread for those 5 hours.

The ERC Stream populates when an analyst clicks GENERATES ERC STREAM. The stream contains the:

  • Average ERC for each date listed.

  • Standard deviation (a measure of dispersion or how widely spread the values are from the mean) for the average daily ERC values.

  • Current ERC value.

The value in the current ERC column displays a "-1" when there is no ERC value from the RAWS/NFDRS. From that point on, assuming the -1 is on, or one day away from the simulation start day, FSPro obtains the ERC value from the forecast. After the specified forecast period (number of days selected) or if no forecast data is used, FSPro uses weather and wind data from the WFDSS-generated time series.

Wind Matrix

Setting up FSPro Winds

The FSPro simulation is greatly influenced by winds, so the user must modify FSPro winds in order to generate good quality analysis data.

For each analysis, FSPro generates a time-series analysis based on historical wind data from the selected Remote Automated Weather Station (RAWS). These historical winds are used for all days outside of any forecast winds. For instance, a 14-day analysis may use 4 days of forecast winds and 10 days from the time series.

As you modify FSPro winds:

  • FSPro, as well as other models in WFDSS, require 20' wind values for inputs. NWS forecasts provide 33' winds. The difference between 20' and 33' winds is negligible and forecasted wind values can be used as is.

  • Select a RAWS representative for the fire area. Consider the wind speeds and directions that influence observed and expected fire behavior. Ask local fire staff or meteorologists which station best represents the winds at the location of the fire. In locations with terrain, it is important to locate a station that is similarly aligned with the topography. Since weather (relative humidity, temperature etc.) is not incorporated into this RAWS selection, the best station for winds may not, necessarily, be the closest to the incident.

  • Ensure that the selected RAWS has sufficient historical data to support the time-series analysis. Some RAWS stations may only have a few years of data available (unlike the ERC station selection 10 years of data is not required). Generally, the more years of data that you can capture for winds the better. When making your RAWS selection, be aware that there could be gaps in the data. You can talk with a local expert and/or download winds to look for these gaps.

  • Test the year filter to determine the optimal timeframe for the historical winds data. The year filter can be set for as long as the station has data. Try different time frames to detect a difference in the wind bins. NOTE: Time frame can be different from time frame from the ERCs.

  • Review the Wind Rose. Winds are binned for computational efficiency. The values in the wind bins are the same as the graphical wind rose. The values in each bin display as percents.

Image T: View Generated Wind Rose at the top of the page under Supporting Information.

  • Select a Winds Type of “10 Min Avg” when modeling “moderate” conditions. Analysts will receive only 10 minute average wind data and no station gust data.

  • Select a Winds Type of “Both” and Generate Wind Matrix when modeling "very high" to "extreme" conditions. Doing so increases wind speeds and includes a greater representation (higher percentages) in the higher wind-speed categories.

  • Select a Winds Type of “Gusts” option uses only gust wind speeds. Use with caution, selecting this option may cause the model to overpredict fire probabilities.

Selecting a Weather Station for Winds

The user must select a weather station to generate the wind matrix, an input required by the FSPRo model to run. The two most important considerations for selecting a weather station that generates a wind matrix are the:

  • Wind matric Weather Station in relation to the local topography and how it influences the recorded winds.

  • Wind Matric Climatology Date Filter to focus on the appropriate winds currently being experienced and predicted.

Select a station that best represents the winds influence for the fire area over its historical record and period of interest. For example, is the station located on a ridge top or valley bottom? Typically, data from weather stations located in valley bottoms reflect up-valley down-valley winds, while ridge-top stations will typically capture winds from 360 degrees, but of generally higher speeds. A station located in a north-south oriented valley bottom which captures only up-valley and down-valley winds will not be appropriate if the winds of concern are primarily from the west/southwest. If needed, you can modify the winds.

 

The drop-down list of available stations contains weather stations from the FEMS database. The closest station (based on a formula for distance and elevation) that is currently reporting observations will display as the default station in the dropdown list.

 

Ask local fire personnel for their input on station selection, view station information and data in Fire Family Plus, or contact the GACC meteorologist for information regarding weather station selection applicability. After you select a weather station for winds, click Notes from the left-side menu list and document why you chose that particular station.

 

CAUTION: The actual weather data available for an individual station is unique to each station. In FSPro, the number of years, months or days within a specific year that actual weather data exist cannot be determined in WFDSS. The months and actual days a station actively collects information is best determined outside the WFDSS system. Knowing this is important when setting the date/time filter's years and start and end months.

  • Fire Streams text

Statusing

WFDSS allows analysts to set the status of the completed analysis runs. There are 5 types of status:

  • Review - Analysis can be viewed, however, it will not be selectable for a decision.

  • Reference - Analysis will be kept for future reference for all analysts on an incident.

  • Calibration - Analysis used to calibrate the model for the final solution.

  • Completed for Strategic Use - Analysis utilized for strategic assessment decisions and inclusion in a decision.

  • Completed for Operational Use - Analysis used to help operational decisions and inclusion in a decision.
